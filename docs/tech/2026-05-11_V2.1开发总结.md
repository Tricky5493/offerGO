## 发现的核心问题

### 本地优化引擎失效（严重）
`runOptimization` 对非技术简历几乎不做任何实质修改：
- `strengthenVerbs`：只处理 `- ` 开头的 bullet points，跳过中文段落和编号列表
- `addQuantPlaceholders`：同上，只处理 bullet points
- `injectAndReorderSkills`：只识别 `技能/技术栈/专业技能` 标题，不识别 `个人能力/核心能力`
- **结果**：优化前后 resume 文本相同，分数不变

### 架构困惑
- Worker 后端（worker.js）已完整实现，但 workers.dev 被墙，国内用户无法访问
- API key + DeepSeek 调用已包含在 Worker 中，但无实际用途

## 待讨论的架构方向

### 方向 A：前端直连 DeepSeek
前端 JS 直接调 `fetch('https://api.deepseek.com')`，不需要 Worker。
- ✅ 国内可达，零后端依赖
- ✅ 最快启动，现在就能用
- ❌ API Key 暴露在前端
- ❌ 无法控制调用频率

### 方向 B：国内云函数做代理
前端 → 阿里云/腾讯云函数 → DeepSeek
- ✅ API Key 安全
- ✅ 可控制频率、加缓存
- ✅ 原生国内服务
- ❌ 多维护一个平台

### 关于 ICP 备案
- **不接支付 = 大概率不需要 ICP**
- 绑 CF 自定义域名可以直接给国内用
- 只有虎皮椒支付才强制需要已备案域名

# 2026-05-14 更新：DeepSeek API 实测与关键决策

## DeepSeek API 实测结果

### 测试条件
- 模型：DeepSeek V3 (`deepseek-chat`)
- 简历：王亦晨（PM/VP，2976 字符，15 年经验）
- 目标方向：产品管理 / 互联网金融 / B2B SaaS
- 成本：**~¥0.003**（2923 tokens）

### 优化效果

| 指标 | 优化前 | 优化后 | 变化 |
|------|------|------|------|
| ATS 预估分 | **65** | **88** | **+23 (+35%)** |
| 新增关键词 | — | +15 个 | — |
| 新增量化指标 | — | +8 个 | — |
| 文本长度 | 2976 字符 | 1166 字符 | -61% |
| 关键数据保留 | — | 14/14 | 100% |

### DeepSeek 做的五件事
1. **结构重组**：段落叙述 → 量化 Bullet Point（ATS 可解析格式）
2. **核心能力提取**：新增 `核心能力` 模块前置（产品战略/商业模式/数据/团队）
3. **动词强化**：负责→主导、完成→推动/实现
4. **关键词注入**：产品战略、数据分析、用户增长、商业化等 ATS 匹配词
5. **去冗余**：删除重复描述，原简历存在大量信息冗余

### 关键发现
- **AI 优化这条路可以走通**：65→88 分是实质性提升，不是数字游戏
- **70% 文本改动 ≠ 70% 事实改动**：所有核心数据点（386亿、120万用户、420亿交易额等）100% 保留
- **本质是"翻译"不是"编造"**：把"跟老板聊的样子"翻译成"大厂 ATS 能读懂的样子"
- **本地规则引擎已无价值**：对于非技术简历，规则引擎永远做不了这种语义级改写

# 2026-05-17 更新 2：CORS 修复 + 超时调整

## 发现的问题

### 1. DeepSeek API 不支持浏览器 CORS 直连
- `fetch('https://api.deepseek.com')` 从浏览器被 CORS 拦截
- `test_optimize.py` 能跑是因为 Python 不走浏览器同源策略
- **结论：前端直连方案（A）不成立，必须走代理**

### 2. 分端口部署导致跨域
- 前端 :8080 + 代理 :8765 → 浏览器视为跨域
- 虽然 CORS 头正确，Chrome 仍可能拦截 localhost 跨端口请求

### 3. 超时设置太短
- 30s 不够 DeepSeek 处理 4600 字符简历（需 20-35s）
- 浏览器 AbortController 先于 DeepSeek 响应触发 → Broken Pipe
- 服务器日志确认：请求到达代理，DeepSeek 正在处理，但浏览器已断开

## 解决方案

### 统一服务器（单端口，零跨域）
```
http://localhost:8765
  ├─ GET  /                → OfferGO_V2.2.html
  ├─ GET  /OfferGO_V2.2.html → 静态文件
  └─ POST /api/optimize     → DeepSeek API 代理
```

### 超时调整
- 浏览器：30s → **90s**
- 服务器：增加 BrokenPipeError 容错

### 安全加固
- API Key 从 V2.2 前端移除
- Key 仅存于 `backend/proxy.py`（服务端）

## 当前状态

```bash
python3 backend/proxy.py

```
