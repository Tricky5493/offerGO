# OfferGO 版本迭代防范机制

## 问题背景

在 V1.4 → V1.5 迭代过程中，多次出现"改新版丢失老功能"的问题：
- screen-optimize 页面丢失
- .loading-card / .company-progress 等 CSS 丢失
- 优化步骤动画 JS 丢失
- goPage() 路由缺失

根本原因：反复覆盖文件 + 增量修改的累积错误，且每次修改后未做全量验证。

---

## 1. 基线原则

| 原则 | 说明 |
|------|------|
| 永不直接覆盖 | 新版本必须从稳定基线 `cp` 创建，不能从零开始 |
| 全量对比 | 每次修改前，对比新旧版本的所有差异 |
| 功能清单 | 维护 checklist，改完逐项打勾验证 |
| 改完必测 | 每次修改后必须走一遍完整用户路径 |

---

## 2. 迭代流程（4步法）

```
Step 1: 基线建立
  └── cp OfferGO_V1.x.html OfferGO_V1.y.html
  └── 确认 V1.y 能正常运行（所有功能 intact）

Step 2: 差异分析
  └── 列出 V1.y 目标功能清单
  └── 标记哪些是新功能，哪些是修改旧功能

Step 3: 增量修改
  └── 每改一个功能，立即验证该功能正常
  └── 改完检查是否破坏了其他功能

Step 4: 回归验证
  └── 对照下方功能清单，逐项测试
  └── 6 个 screen 全部走一遍完整流程
```

## 3. 功能清单（每次迭代前复制并逐项验证）

### 3.1 六个 Screen

- [ ] screen-landing（首页：数据卡片 + 上传区域）
- [ ] screen-loading（诊断加载：3步动画）
- [ ] screen-score（评分报告：各厂匹配度 + 敬请期待）
- [ ] screen-optimize（优化加载：5步动画）
- [ ] screen-result（对比结果：定价区域）
- [ ] screen-user（用户中心：统计 + 菜单）

### 3.2 关键 CSS 组件

- [ ] .stat-card / .stat-num / .stat-lbl（首页数据卡片）
- [ ] .upload-card / .upload-area（上传区域）
- [ ] .loading-card / .loading-spinner（加载页）
- [ ] .company-progress / .cp-item / .cp-bar / .cp-fill（步骤进度条）
- [ ] .co-grid / .co-card（各厂匹配度卡片）
- [ ] .coming-soon-section（敬请期待区域）
- [ ] .paywall / .pay-tier（定价区域）
- [ ] .bottom-nav / .nav-item（底部导航）
- [ ] .user-header / .user-stats / .menu-list（用户中心）

### 3.3 关键 JS 函数

- [ ] show()（V1.4 原生页面切换）
- [ ] goPage()（V1.5 新增，支持底部导航）
- [ ] 诊断流程：上传 → screen-loading → screen-score
- [ ] 优化流程：screen-score → screen-optimize → screen-result
- [ ] 底部导航切换：首页 ↔ 我的
- [ ] resetAll()（重新上传）

### 3.4 测试路径

```
路径1: 首页 → 上传简历 → 诊断加载动画(3步) → 评分页
路径2: 评分页 → 开始优化 → 优化加载动画(5步) → 结果页
路径3: 底部导航 → 首页 ↔ 我的（双向切换）
路径4: 结果页 → 重新上传 → 回到首页
```

## 4. 代码审查 Checklist（改完后必做）

| 检查项 | 命令 |
|--------|------|
| 6 个 screen 都存在 | `grep -c 'id="screen-' file.html` → 应输出 6 |
| 关键 CSS 类有定义 | `grep '\.loading-card{' file.html` |
| 没有重复 ID | `grep -o 'id="[^"]*"' file.html \| sort \| uniq -d` |
| 关键函数有定义 | `grep 'function show\|function goPage' file.html` |
| SCREENS 数组完整 | `grep 'var SCREENS' file.html` |
| 没有 HTML 语法错误 | 浏览器 F12 → Elements 面板检查 |

---

## 5. 常见陷阱

| 陷阱 | 正确做法 |
|------|---------|
| 从已损坏的版本继续改 | 从稳定基线（V1.4）重新 cp |
| 只改 HTML 忘了改 CSS | 每个功能块必须 HTML+CSS+JS 一起迁移 |
| 只改 CSS 忘了改 JS | 同上 |
| SearchReplace 匹配到错误位置 | 替换前先 Read 确认上下文 |
| 改完不验证 | 必须走一遍完整用户路径 |
| 浏览器缓存导致误判 | 用无痕窗口或 Ctrl+Shift+R 强制刷新 |

---

## 6. 版本命名规范

```
OfferGO_V1.x.html     ← 稳定发布版
OfferGO_V1.x.1.html   ← 小修（bugfix）
OfferGO_V1.x+1.html   ← 大版本（新功能）
```

永远不要覆盖已发布的稳定版本文件。
