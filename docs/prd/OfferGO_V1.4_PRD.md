## 1. 产品目标（3个清晰正交的目标）

### 目标1：跑通商业闭环
接入真实AI API实现简历解析与ATS评分，接入微信支付完成真实交易，构建用户账户体系实现留存。**将V1.3的Demo演示转化为可运营的真实产品**，实现从"小红书曝光→上传简历→ATS诊断→付费优化"的完整商业化链路。

- **北极星指标（验证期）**：完成ATS诊断数（日活诊断数）
- **成功标准**：月付费用户100+，付费转化率>2%

### 目标2：建立"大厂ATS评分"品类独占认知
以"行业唯一6大厂独立ATS评分引擎"为核心差异点，在小红书通过评分截图、优化对比等社交素材建立品牌心智。对标"大厂ATS评分=OfferGO"的品类联想，形成竞争壁垒。

- **北极星指标（增长期）**：周活跃付费用户数
- **成功标准**：小红书自然搜索"ATS评分"相关笔记中，OfferGO品牌提及率>60%

### 目标3：构建数据驱动的增长飞轮
通过用户上传的简历数据积累，持续优化ATS评分算法的准确性和关键词库，形成"数据积累→算法提升→用户体验更好→更多用户"的正向飞轮。同时建立完整的数据埋点体系驱动产品迭代。

- **北极星指标（长期）**：完成ATS优化并进入面试流程的用户数
- **成功标准**：用户留存率（D7）>15%

## 2. 用户故事（5个关键场景）

### 故事1：应届生的第一次"惊吓"（画像A · 转化核心场景）
> 小红书上看到室友分享的OfferGO评分截图——"字节跳动34分"。小明（22岁，985研三，秋零offer）好奇地扫码进入，上传自己的简历，30秒后看到各家大厂评分：字节28分，阿里45分，腾讯39分。他慌了，看到"AI优化预估可提升至82分"，点击优化按钮。优化完成后预览看到"量化指标+4组，大厂关键词+7个"，对比从34分飙到85分。支付9.9元下载优化版PDF。

### 故事2：跳槽人的精准匹配（画像B）
> 小张（28岁，中厂后端，4年经验）刷小红书时看到"你的简历适合哪家大厂"的测试帖，点进去看到OfferGO。他上传简历后，系统不仅给出6厂评分，还在详情页标注了每家缺什么关键词："字节缺《高并发/用户增长》，阿里缺《数据驱动/AB测试》"。他选了字节和阿里两个目标，支付49.9元开通年费会员，一次优化两个厂。

### 故事3：被裁后的批量投递（画像C）
> 王姐（31岁，产品经理，被裁）日投50家，效率为王。她需要快速批量优化简历适配不同岗位。上传简历后选择"产品经理"方向，系统自动推荐5个大厂的JD匹配分析。支付9.9元单次优化后效果明显，但她发现多个岗位都要投，升级为年费会员。

### 故事4：免费→付费的犹豫感（裂变入口）
> 小李收到OfferGO的分享卡片"测测你的大厂ATS分数"，免费做了一次诊断，看到评分42分。结果页面展示"已有12,847人使用"的滚动计数器，同时弹出限时优惠"¥9.9解锁完整版（原价¥49.6）"。犹豫中看到"你的同学XX已使用"的社交证明，最终付费。

### 故事5：次日唤醒与复购（留存场景）
> 前一天付费下载了字节优化版简历，App次日推送"¥9.9再优化一个厂，今日第二份半价"。用户回到产品，发现历史记录还在，无需重新上传，直接选择阿里继续优化。

## 3. 用户研究洞察

### 3.1 核心画像聚焦

| 画像 | 身份 | 焦虑等级 | 价格敏感度 | 核心场景 | 推荐策略 |
|------|------|:--------:|:----------:|----------|----------|
| A · 应届生 | 22-25岁，零offer | 9/10 | 高（¥4.9-19.9） | 秋招季，看到同辈有offer | 低价单次→年费升级 |
| B · 跳槽人 | 26-30岁，在职 | 6/10 | 中（¥9.9-49.9） | 想涨薪30-50%，骑驴找马 | 直接推年费（性价比高） |
| C · 被裁重启 | 28-35岁，被裁 | 8/10 | 高（¥4.9-29.9） | 紧急求职，海投 | 单次体验→7天无限制 |

### 3.2 付费心理：四大驱动力

1. **损失厌恶**（最强驱动力）：用户看到34分的ATS评分→"不付费=继续用烂简历投递"→焦虑驱动立即决策
   - **关键设计**：评分页突出"低分警示"，优化前后对比分数差要醒目
2. **社会证明**："已有12,847人使用"、"上周通过OfferGO优化拿到面试的人数"
   - **关键设计**：全页底部固定滚动条，评分页、付费页多处复用
3. **低价锚定**：¥9.9 vs ¥49.9 vs ¥99 形成价格锚定效应
   - **关键设计**：¥49.9年费在¥9.9旁边显得"超值"，¥99的1v1服务衬托AI优化的性价比
4. **紧迫感**：秋招倒计时 / 大厂投递DDL
   - **关键设计**：如果检测到当前是招聘季，头部显示倒计时组件

### 3.3 信任Gap与解决方案

**核心问题**：用户不相信"AI优化能真正有效"。

**解决方案设计**（产品内需全部落实）：

1. 优化前后对比展示"每处改动基于哪家大厂JD要求" → 已实现V1.3的diff高亮UI
2. 每条优化建议标注出处："根据字节跳动2026届后端JD，关键词'高并发'命中率要求>85%" → **P1需求**
3. 展示优化逻辑透明度："系统建议新增的7个关键词中，5个出现在目标JD中" → **P1需求**
4. 首单免费体验完整流程（但只预览前3行优化结果） → **P0需求**

### 3.4 用户旅程关键点

```
曝光（小红书笔记）→ 扫码/搜索 → 打开OfferGO（H5）→ 上传简历
→ ATS评分（看到低分，焦虑产生）→ 点击优化
→ 看到优化效果（分数提升）→ 预览片段 → 付费下载
```

**需要重点优化的环节**：
- 上传环节：支持PDF/Word/图片/粘贴文本，降低上传门槛
- 评分环节：评分可信度是第一关，需要展示评分依据而非"黑盒"
- 付费环节：微信支付直连，<3秒完成，支持指纹支付

## 6. 需求池

### P0 - 商业化基础设施（必须做，否则无法运营）

#### 需求1：接入AI API实现真实简历解析与ATS评分

- **背景**：V1.3所有评分数据为硬编码Demo，用户上传简历后无真实处理。这是从Demo到产品的关键转折点。
- **验收标准**：
  1. 用户上传PDF/Word简历后，系统返回结构化简历文本（个人资料、工作经历、技能标签）
  2. 系统根据简历内容，针对6家大厂（字节/阿里/腾讯/美团/京东/百度）分别计算ATS匹配分数
  3. 评分维度覆盖：关键词覆盖率、量化指标密度、技能匹配度、格式友好度
  4. 评分结果返回时间<30秒（含API调用）
- **工作量**：1-2周
- **技术建议**：对接大模型API进行简历解析，配合规则引擎做ATS评分

#### 需求2：接入微信登录+用户账户系统

- **背景**：V1.3无任何用户体系，纯匿名使用，无法留存用户、无法追踪付费状态、无法复购。
- **验收标准**：
  1. 用户首次使用弹窗"微信一键登录"（静默授权，用户无感）
  2. 登录后生成userId，保存用户上传历史、付费记录、优化记录
  3. 用户再次访问时自动登录（基于微信OAuth），无需重复授权
  4. 后端提供用户数据CRUD接口
- **工作量**：1周
- **技术建议**：微信公众平台OAuth2.0静默授权 + 简单的用户表（userId, wxOpenId, nickName, avatarUrl, createdAt, lastLoginAt）

#### 需求3：接入微信支付

- **背景**：V1.3付费流程为 `tip() → alert("付费功能演示 — 接入支付即可上线")` ，完全不可用。
- **验收标准**：
  1. 用户点击付费按钮后，唤起微信支付弹窗（JSAPI支付）
  2. 支付成功后回调更新用户付费状态
  3. 支付失败/取消有明确提示和处理
  4. 支持¥9.9、¥19.9、¥49.9、¥49四个SKU
  5. 用户已购买项目不重复收费
- **工作量**：1周
- **技术建议**：微信JSAPI支付 + 简单的订单表

#### 需求4：简历上传流程优化（多格式支持）

- **背景**：V1.3支持 PDF/Word/图片，但仅展示文件名，无真实内容解析，且不支持文本粘贴。
- **验收标准**：
  1. 支持PDF（.pdf）、Word（.doc/.docx）、图片（.png/.jpg/.jpeg）三种格式上传
  2. 支持文本粘贴（兼容无法上传文件的场景，如电脑端）
  3. 上传失败率<5%（含格式不支持、文件过大>10MB等情况）
  4. 上传过程有进度条/加载状态
  5. 非法文件（病毒/非简历文件）有校验和提示
- **工作量**：3天

### P1 - 核心功能增强（提升转化率）

#### 需求5：岗位方向选择（而非仅选公司）

- **背景**：V1.3用户只能选择目标大厂，但不同岗位的ATS要求差异巨大，同一个公司在不同岗位上的评分完全不同。
- **验收标准**：
  1. 用户上传简历后，先选择"岗位方向"（后端开发 / 前端开发 / 产品经理 / 数据分析 / 运营 / 设计 / 算法 / 测试 等8个方向）
  2. 系统根据岗位方向匹配对应JD库进行ATS评分
  3. 评分结果页面标注"该岗位方向+目标公司"的复合匹配度
- **工作量**：3天

#### 需求6：单厂评分详情页（关键词匹配明细）

- **背景**：V1.3仅有综合评分和简短的评分描述，用户无法理解"为什么我的分数这么低"。这是信任Gap的核心来源。
- **验收标准**：
  1. 点击单厂评分卡片，进入详情页
  2. 详情页展示：该厂该岗位的TOP20关键词列表
  3. 标记简历中"已匹配的关键词"和"缺失的关键词"
  4. 显示"你的简历命中率"（已匹配数/总需求数）
  5. 支持关键词点击查看JD原文出处
- **工作量**：4天
- **优先级依据**：瑞思研究明确"信任Gap是最核心问题，关键词明细是解决方案"

#### 需求7：优化前后对比截图生成（社交传播素材）

- **背景**：这是小红书获客的核心驱动力。V1.3已有对比UI但无法生成长图分享。
- **验收标准**：
  1. 优化完成后，用户可一键生成"优化前后对比长图"
  2. 长图包含：优化前分数 vs 优化后分数、分数提升幅度、"由OfferGO优化"品牌水印
  3. 长图自动适配手机屏幕比例，可直接保存到相册
  4. 分享按钮文案：引导用户发小红书（"晒出你的大厂ATS分数"）
- **工作量**：2天
- **优先级依据**：竞析建议"ATS雷达图可视化是传播素材"，这是成本最低的增长渠道

#### 需求8：免费版限次策略（首次免费，深度付费）

- **背景**：V1.3所有功能完全免费（包括无限优化），毫无付费动机。需要重构免费/付费权益边界。
- **验收标准**：
  1. 首次访问：免费获得完整ATS诊断+评分+问题列表
  2. 优化结果预览：仅展示前3行优化内容（"付费预览片段"）
  3. 完整优化结果/PDF下载：需付费
  4. 免费诊断次数：每个微信用户限1次免费诊断，之后需要付费
  5. 界面清晰标注"免费" vs "Pro"功能差异
- **工作量**：2天

#### 需求9：付费预览片段（优化结果前3行免费展示）

- **背景**：用户在付费前需要确认"这个优化真的有效吗"。预览片段既展示了价值，又保留完整版作为付费门槛。
- **验收标准**：
  1. 优化完成后，默认展示优化结果的前3行（约60字）
  2. 3行之后的内容模糊/隐藏/显示"+N处改动已隐藏，立即解锁完整版"
  3. 免费预览区包含至少1处实质性改动（非空洞描述），让用户感知价值
  4. 付费后自动展开全部内容
- **工作量**：1天

#### 需求10：社会化证明组件（"已有xxxx人使用"）

- **背景**：瑞思研究确认社会证明是付费驱动力的第二名（仅次于损失厌恶）。
- **验收标准**：
  1. 评分结果页、付费页底部显示滚动计数器："已有 xx,xxx 人使用"
  2. 数字每日零点自动更新（基于数据库真实注册用户量）
  3. 同时支持展示"今日优化 xxx 份简历"
  4. 可选展示"上周通过优化拿到面试机会 x 人"
- **工作量**：1天

### P2 - 增长与粘性功能

#### 需求11：AI面试模拟（基于简历生成面试题）

- **背景**：用户优化简历后，下一步自然需求是准备面试。这是提升用户粘性的关键功能。
- **验收标准**：
  1. 基于用户简历内容和目标公司/岗位，AI生成5道高频面试题
  2. 每道题提供参考答案思路
  3. 面试题可保存/分享
  4. 支持"语音回答→AI评估"（高级功能，可推后）
- **工作量**：3天

#### 需求12：投递日历/求职进度追踪

- **背景**：求职是一个过程管理问题，用户需要记录投递进度。
- **验收标准**：
  1. 用户可记录投递公司、岗位、日期、状态（已投/简历筛选/笔试/面试/Offer/挂）
  2. 日历视图展示投递节奏
  3. 每日提醒"今日投递计划"
  4. 全流程数据看板（投递总量、面试率、Offer率）
- **工作量**：5天

#### 需求13：社交裂变分享（"测测你的大厂ATS分数"）

- **背景**：利用用户的好奇心和社会比较心理实现病毒传播。
- **验收标准**：
  1. 用户完成诊断后，可生成"ATS分数测试卡"分享到微信/小红书
  2. 卡片展示用户的ATS分数 + "你的ATS分数超过xx%的人"
  3. 新用户通过卡片进入后，走完整诊断流程
  4. 分享卡片带参数追踪，统计裂变系数K
- **工作量**：2天

#### 需求14：招聘季倒计时组件

- **背景**：秋招/春招是应届生的核心焦虑源，紧迫感驱动付费。
- **验收标准**：
  1. 首页顶部检测当前是否是招聘季（秋招8-10月/春招3-4月）
  2. 招聘季期间显示倒计时："2026秋招倒计时 X 天"
  3. 倒计时下方显示"大厂DDL汇总"（字节8月31日截止/阿里9月15日等）
  4. 非招聘季显示"距离下个招聘季 X 天"
- **工作量**：1天

#### 需求15：关键词热词推荐系统

- **背景**：利用用户上传的简历数据积累，分析各厂各岗位的高频命中关键词，形成数据飞轮。
- **验收标准**：
  1. 基于已有简历数据，统计每个大厂+岗位方向的TOP30关键词
  2. 在评分详情页展示"该岗位热门关键词"推荐
  3. 关键词展示匹配度（该关键词出现在多少通过优化的简历中）
  4. 用户可一键复制关键词到简历修改
- **工作量**：3天（后端算法）+ 1天（前端展示）

## 8. 用户旅程与关键流程

### 8.1 完整用户旅程图

```
【第一阶段：获客】
小红书信息流 → 看到ATS评分截图笔记 → 产生好奇
    ↓
小红书搜索"ATS评分" → 看到多条OfferGO相关笔记
    ↓
扫码/点击链接 → 进入OfferGO H5页面

【第二阶段：诊断】
首页 → 上传简历（PDF/Word/图片/粘贴文本）
    ↓
微信一键登录（静默授权）
    ↓
选择岗位方向（8选1）
    ↓
AI解析简历 → 6大厂ATS评分计算（<30s）
    ↓
诊断报告页：
  ├─ 综合评分（仪表盘）
  ├─ 各厂评分卡片（6个）
  ├─ 问题列表（关键词/量化/格式）
  └─ 岗位方向评分详情（可点击展开）

【第三阶段：转化】
看到低分 → 焦虑 → 点击"AI一键优化"
    ↓
选择目标大厂（1-3个）
    ↓
优化loading动画（展示优化步骤）
    ↓
优化完成 → 优化前后对比（分数提升）
    ↓
付费预览页：
  ├─ 完整评分对比仪表盘
  ├─ 优化后简历预览（前3行）
  ├─ 关键改动摘要（+N个关键词 / +N组数据）
  ├─ 社会证明组件
  └─ 付费选项（¥9.9 / ¥19.9 / ¥49.9 / ¥49）

【第四阶段：付费】
选择SKU → 微信支付 → 支付成功
    ↓
解锁完整优化结果 → 可查看全部改动
    ↓
一键生成对比截图 → 分享到小红书
    ↓
下载优化后PDF → 直接投递

【第五阶段：留存】
第2日推送 → "再优化一个厂，今日第二份半价"
    ↓
打开OfferGO → 历史记录可见 → 选择其他大厂
    ↓
优化完成 → 投递 → 进入面试
    ↓
使用AI面试模拟功能（P2）
```

### 8.2 关键流程细节

**流程A：ATS诊断流程（核心主线）**

```
上传简历 → 文件校验（格式/大小） → 
  ├─ 失败 → 提示重新上传（支持格式说明）
  └─ 成功 → 调用AI解析API → 
      ├─ 解析失败 → 提示"简历格式复杂，建议粘贴文本"
      └─ 解析成功 → 提取结构信息 → 
          并行调用6厂评分引擎 → 聚合评分结果 → 
          生成诊断报告 → 渲染前端UI
```

**流程B：付费优化流程**

```
点击"优化" → 选择目标岗位方向 → 选择目标大厂（1-3个）→
  ├─ 免费用户且非首次 → 跳转付费提示 →
  │   ├─ 未付费 → 返回
  │   └─ 付费 → 继续
  └─ 免费首次 / 已付费用户 → 
      调用AI优化API → 生成优化结果 → 
      展示对比页面 → 付费预览（前3行）→
      用户点击付费 → 调到微信支付 → 
      ├─ 失败 → 重试
      └─ 成功 → 解锁全部内容
```

**流程C：二次唤醒流程**

```
用户未付费退出 →
后台记录用户userId + 当前诊断结果 →
24h后触发模板消息推送 →
"你的大厂ATS评分还在，但已有xx人已抢先优化" →
用户点击 → 回到结果页 →
显示"限时优惠：¥4.9体验价（原¥9.9）" →
用户付费 / 关闭
```

## 9. Non-goals（明确不做什么）

| 功能 | 不做原因 | 可能规划 |
|------|----------|----------|
| **AI写简历（从零生成）** | 与现有差异化定位冲突，资源投入大，非核心用户需求 | V1.8+ 考虑 |
| **求职社区/论坛** | 社区运营成本高，需要内容和审核团队 | V2.0+ |
| **面试题库（海量）** | 市场已有大量面试工具，不做同质化竞争 | 仅做AI面试模拟（基于简历） |
| **PC/多端同步** | 初期聚焦移动端H5，小屏幕场景以"扫码即用"为核心 | V1.6+ |
| **批量投递自动化** | 涉及招聘平台接口，合规风险高 | 不不规划 |
| **企业端/HR端** | 初期B2C模式，不做B端 | 不做规划 |
| **简历模板市场** | 竞品模板已规模化，竞争不过 | 仅基础模板作为会员权益 |
| **英文简历优化** | 初期聚焦国内大厂中文场景 | V1.8+ |

**关于"不做"的原则**：V1.4的核心目标是在最小可行范围内跑通"上传 → 诊断 → 优化 → 付费"闭环，所有分散资源的功能都不做。
